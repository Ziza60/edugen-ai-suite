# PPTX Engine v5 — Version History (archive)

This file archives detailed version notes for the PPTX engine v5 (`supabase/functions/export-pptx-v4`).
The current version + active architecture lives in `replit.md`. Older entries are kept here for context
when investigating regressions or understanding why specific guards / repairs exist.

- **v5.5.8** — **Code synthesis priority inversion** (stop demoting code slides, start completing them). User report: v5.5.5-v5.5.7 silently converted CODE slides to bullets/concepts whenever repair failed (slide 14 perdeu snippet, slide 20 virou bullets, módulo 7 perdeu unittest/logging/pdb). Fix is architectural: synthesize a minimal complete example from module context BEFORE demoting; demote only as last resort with explicit `[CODE-SLIDE-CONVERSION]` log. (1) **`synthesizeMinimalCompleteExample(moduleKind, ctx, slideTitle)`** new function (~115 lines) returns `{code, source}` or `null`. Per-kind canonical snippets: `oop` (reuses `ctx.classes[0]`+`ctx.vars[0]`+first non-`__init__` method when available, else canonical `Carro` class with `exibir_info()`); `tests_logs` (title-routed to `pdb.set_trace`/`logging.basicConfig+info+warning`/`unittest.TestCase` with `assertEqual`); `json_apis` (`requests.get`+`status_code==200`+`response.json()`); `files_exceptions` (`with open` + `try/except FileNotFoundError`); `control_flow` (`if/elif/else` `classificar(nota)`); `data_structures` (list of dicts + `sum(p["preco"] for p in ...)`); `fundamentals` (`input()`+`print(f"...")`); `best_practices` (typed function with docstring). Source label `module_ctx` when reusing accumulator symbols, `canonical_<kind>` for fallback. (2) **Guardrail 2 INVERTED**: `layout=code` with empty code → call `synthesizeMinimalCompleteExample(moduleKind, prevSymbols, out.title)` FIRST. If success → assign synth code + `[CODE-SYNTHESIS] reason=empty_code beforeLines=0 afterLines=N source=…`. Only if `null` → emit `[CODE-SLIDE-CONVERSION] from=CODE to=CONCEPT reason=empty_code_no_synth_available` THEN demote. (3) **Guardrail 4 INVERTED**: when `repairIncompleteCodeExample` returns `null` AND layout=code → try synth; on success replace code + log `[CODE-SYNTHESIS] reason=repair_failed (<reason>)`. Only if synth also `null` → `[CODE-SLIDE-CONVERSION]` + demote. (4) **Guardrail 5 INVERTED**: code-slide integrity fail (codeLines<3 or no syntax) → try synth before demote. (5) **Logs added**: `[CODE-SYNTHESIS]` (success path with `beforeLines`/`afterLines`/`source`/`moduleKind`), `[CODE-SLIDE-CONVERSION]` (failure path with `from=CODE to=CONCEPT reason=…`). User-spec compliance: every CODE→CONCEPT transition is now logged with explicit reason — no more silent demotes. **Constraint preserved**: `repairIncompleteCodeExample` still runs first (fast path); synth is the second-chance layer. Renderer/planner/preservation/qaVeto/billing/route/schema unchanged. `ENGINE_VERSION = "5.5.8"`.

- **v5.5.6** — **Context-aware code repair (Manus-inspired flow)**. v5.5.5 saw repair work but in isolation: a slide could be "completed" with names disconnected from the module's earlier vocabulary. Three additions: (1) **`CodeSymbols` type widened** to `{classes, vars, funcs, imports}` (was `{classes, vars}`); `extractCodeSymbols` now also captures `def funcname(` and `from X import` / `import X`. (2) **`extractSemanticCodeContext(slides[])`** walks an array of accepted slides and returns the deduplicated UNION of all symbols across them — preserving order so `vars[i]` corresponds to `classes[i]` when both came from the same line (`livro = Livro("Python")` → classes[0]="Livro", vars[0]="livro"). (3) **Module accumulator wired in dedup loop**: `moduleCtx` replaces single-slide `prevSymbols`. After each kept slide → `moduleCtx = extractSemanticCodeContext(kept)`. So slide 5's repair sees ALL classes/vars introduced in slides 1-4 of the same module, not just slide 4. (4) **Drift guard inside `repairIncompleteCodeExample`**: when a slide defines `class X` AND `moduleCtx.classes` is non-empty AND `X` is NOT in `moduleCtx.classes` → refuse repair entirely (`return null`) and log `[CODE-CONTEXT-DRIFT] reason="class 'X' is new vs module context [Livro, ...]" action=refuse_repair`. Let Guardrail 4 demote layout=code to bullets — better than baking in a contradicting domain. (5) **Instance reuse from context**: when class IS in moduleCtx, repair looks up `moduleCtx.vars[ctxInstIdx]` to get the original instance name (e.g. `livro`) rather than synthesizing `cls.toLowerCase()`. So slide N+1 completing `class Livro` reuses `livro` from slide N. (6) **`[CODE-CONTEXT]` log on every successful class-demo repair**: `slide=N reusedClasses=["Livro"] reusedVariables=["livro"] newSymbols=[] contextualConsistency=PASSED`. `applyFinalGuardrails` signature now takes `prevSymbols: CodeSymbols` (was the narrow 2-field shape). No renderer / planner / preservation / qaVeto / billing / route / schema changes. `ENGINE_VERSION = "5.5.6"`.

(7) **`isGenericObjective` looser pattern** in presentation-plan.ts: ANY of `compreender|entender|aplicar|identificar|conhecer|estudar|aprender|explorar|analisar|examinar` followed by gerund glue (`trabalhando|estudando|explorando|lidando|atuando|operando`) + `com|em|sobre` is generic regardless of suffix match.
