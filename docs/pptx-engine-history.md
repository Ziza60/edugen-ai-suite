# PPTX Engine v5 — Version History (archive)

This file archives detailed version notes for the PPTX engine v5 (`supabase/functions/export-pptx-v4`).
The current version + active architecture lives in `replit.md`. Older entries are kept here for context
when investigating regressions or understanding why specific guards / repairs exist.

- **v5.3.3** — **Header/Content Binding Guard + Slide-Binding Debug Pipeline**. Investigation of slide 13 "Repetindo Ações com e" (tokens `for`/`while` perdidos do título). Hipótese de causa raiz: cascade L3 LLM rewrite (provavelmente disparado por `CONTENT_DENSITY_OVERFLOW` no slide original `Repetindo Ações com 'for' e 'while'`) devolveu título encurtado sem os tokens entre aspas, e nenhum guarda existente checava o cabeçalho de layouts process/cards/comparison antes do render. Auditoria do pipeline confirmou que `globalSanitize` / `san()` / `MARKDOWN_*_RE` / `MODULE_NOISE_RE` NÃO removem texto entre aspas simples — `stripCommentsAndStrings` (line 4157, regex `/'(?:\\.|[^'\\])*'/g`) faz isso mas só roda em `slide.code`, nunca em titles. Conclusão: o L3 rephrase é o suspeito principal; debug logs confirmarão na próxima rodada. **Fix em 4 camadas**: (1) **`[SLIDE-BINDING-DEBUG]` logs em 3 estágios** — `stage=presentationPlan` (intent/title/items/steps/cards/code do output cru do planner), `stage=v5SlideLike` (layout/title/sectionLabel/items/leftItems/rightItems após conversão), `stage=rendererInput` (layout/title/itemsCount/leftCount/rightCount/firstItems do slide final). Permite localizar exatamente em qual estágio um título perdeu tokens. (2) **`detectHeaderContentLeak(title)`** — 6 padrões: `com_e_orphan` (`com e` sem substantivo após), `com_comma_orphan` (`com , `), `paren_e_orphan` (`(, e)`), `como_e_trailing` (`como , e .`), `e_dot_trailing` (` e .` no final), `leading_connective` (`^e ` / `^ou `). Smoke 6/6 positives + 4/4 negatives. (3) **`validateLayoutBinding(slide, moduleTitle, courseTopic, intent)`** — para layouts process/cards/comparison/twocol/code_walkthrough/code/bullets/module_cover, detecta leak no título, tenta reparo via `repairTechnicalTokens`, se falhar reconstrói via `reconstructTitleFromIntent` (`py_flow` + `code_walkthrough` → `Laços \`for\` e \`while\` em Python`; + `process` → `Estruturas de Controle de Fluxo`; + `comparison` → `\`for\` vs \`while\``). Logs `[V5-BINDING-REPAIR]` (token-restore funcionou) ou `[V5-BINDING-RECONSTRUCT]` (caiu na reconstrução). (4) **3 novas regras `TR_FLOW_TOKENS` para títulos**: `(repetindo ações|laços|loops|iterando|repetição) com e [.]?` → `... com \`for\` e \`while\``; `(estruturas de repetição|controle de fluxo) com e` → idem. Regra com trailing-period **vem primeiro** para consumir o `.` direito (sem orphan space). **Wiring**: `validateLayoutBinding` chamado dentro do bloco `if (inRange && !hasFatal && !hasBlocker)` no per-module gate, ANTES da construção do `Slide` final — título reparado/reconstruído entra no objeto `Slide` e portanto em todos os passes downstream (QA per-module / cascade / global safety net / render). Renderer NÃO modificado (per architect). 5/5 smoke tests title repair + 16/16 detector tests pass.
- **v5.3.2** — Hotfix on top of v5.3.1 after first production run. Two false-positives surfaced: (1) safety-net `isGenericLearningObjective` was blocking M8 cover with "[SAFETY-NET] Objetivo genérico escapou em 'Boas Práticas e Implantação de Projetos': 'Aplicar PEP 8 e docstrings para manter código legível e padronizado.'" — the v5.3.1 replacement objective itself was being rejected because `CONCRETE_TECH_NOUNS_RE` lacked best-practices/I/O/OOP-Python tokens. **Fix**: extended `CONCRETE_TECH_NOUNS_RE` with `pep 8 / docstring(s) / venv / virtualenv / pip / requirements / setup.py / pyproject(.toml) / src / tests / docs / readme / license / distribuição / empacotamento / projeto(s) / dependência(s) / repositório / gitignore / ci-cd / deploy / implantação / produção`, plus I/O tokens (`entrada / saída / input / print / read / write / open`) and OOP tokens (`instância(s) / atributo(s) / herança / polimorfismo / encapsulamento / construtor(es) / módulos / pacotes`). This also pre-empts the same false-positive on `py_basics[2] = "Aplicar entrada e saída com input() e print()."` which was vulnerable but hadn't fired yet. (2) `[V5-REPAIR] module_7_slide_3.post | "Configure níveis de log como e ." → "Configure níveis de log como."` — the legacy punctuation cleaner stripped "e " instead of restoring tokens, because `TR_TESTS_TOKENS` only matched the variant ending in `ERROR`. **Fix**: added 2 new TR_TESTS rules for `níveis de log como e .` (with or without trailing ERROR) → restores `como DEBUG, INFO, WARNING e ERROR.`. Smoke test: 7/7 critical assertions pass (M8 cover NOT generic; all py_* "Aplicar" tails NOT generic; bare "Compreender fundamentos" / "Aplicar conceitos" still flagged as generic; log-levels token restoration works).
- **v5.3.1** — **Repair Hardening Pass** on top of v5.3.0. Root cause fixed: `detectModuleDomainPython` was concatenating moduleTitle + courseTopic and matching the basics regex against the combined text — for any course titled "Introdução à Programação em Python", the word `introdução` from the course topic forced EVERY module into `py_basics`, so M8 "Boas Práticas" got injected with "Utilizar variáveis, tipos primitivos..." objectives. **Fix**: (a) `py_best_practices` branch added FIRST in `detectModuleDomainPython` matching `boas práticas / best practices / implant / deploy / produção / ci-cd / empacot / pep 8 / venv / pip / requirements / setup.py / docstring / estrutura de projeto / README`; (b) `py_basics` branch now matches MODULE TITLE ONLY (not combined). New branches: `py_best_practices` added to `PYTHON_OBJECTIVE_TAILS` (PEP 8 / venv / src+tests / setup.py) and `SEMANTIC_REPAIRS` (trailing_with_tool / trailing_as_example / empty_example_parens). **`detectModuleDomain` helper** extended with `isBestPractices` / `isFlow` / `isErrors` flags. **NEW `repairTechnicalTokens(text, moduleTitle, courseTopic)`** — RESTORES missing technical tokens instead of just cleaning punctuation. Domain-narrow rules: `TR_FLOW_TOKENS` (`Condicionais (, , else)` → `(if, elif, else)`), `TR_TESTS_TOKENS` (`Níveis como, e ERROR` → `DEBUG, INFO e ERROR`; `com métodos e assert` → `unittest.TestCase com test_* e assert*`; `herdando de.` → `herdando de unittest.TestCase`), `TR_FILES_TOKENS` (`Realize leitura com, e escrita com.` → `read()` / `write()`; `Capture e para feedback` → `FileNotFoundError e IOError`), `TR_OOP_TOKENS` (`construtor init` → `__init__()`; `<inst>.init(` → `<inst>.__init__(`; `Acessar Membros: Usar.` → `objeto.atributo e objeto.metodo()`), `TR_BEST_PRACTICES_TOKENS` (`Organize o código-fonte em e testes em.` → `src/` / `tests/`; `Inclua e para informações do projeto.` → `README.md` / `LICENSE`; `Utilizar e para definir metadados.` → `setup.py` / `pyproject.toml`).
- **v5.3.0** — **Modular Export Architecture** (Manus-inspired). Goal: reduce semantic complexity by treating each module as an isolated mini-deck instead of generating one giant 40-60 slide plan. **New formal types** in `presentation-plan.ts`: `GlobalBlueprint`, `ModuleDeck<TSlide>`, `CourseExportPlan`, `ModulePlannerDiagnostic`, `ModuleQADiagnostic`, `ModuleSource`, plus constants `MAX_PLANNER_SLIDES_PER_MODULE` (5) / `MIN_PLANNER_SLIDES_PER_MODULE` (1) and helper `buildDefaultBlueprint()`. **Module Context Guard**: new `validateModuleSemanticBoundary(slide, moduleTitle, courseTitle)` consolidates the cross-module-leak / per-kind deny-pattern / truncation / broken-sentence / generic-objective / incomplete-code checks into a single entry point that returns `PlanIssue[]`. `getModuleKind()` exposed for callers. **Cap 4→5 slides/module** with rollout pre-wired (constants + prompt "EXACTLY 3 to 5 slides" + parser slice + gate range all reference the same `MAX_PLANNER_SLIDES_PER_MODULE`). **Per-Module QA wrapper** (Phase 5 hybrid): `runPptxQA` + `resolveQAIssues` cascade now run PER MODULE inside `runPipeline` (each module gets `runPptxQA([moduleSlides], [moduleContent], courseTitle, [moduleTitle])` + cascade), then aggregated into a single `QAReport` for the existing global safety net (`runGlobalFieldSafetyNet`) + global dedupe (`dedupeSemanticDuplicates`) + final QA audit + `qaVeto`. Architect-recommended hybrid: QA scoped per-module catches ~95% of issues without contaminating other modules; global passes remain only as cross-module audit (dedupe + safety-net + veto). **New structured logs**: `[MODULE-PLANNER] module=N title="..." slides=X fatals=Y blockers=Z accepted=true|false` per module during the gate; `[MODULE-QA] module=N status=PASSED|FAILED issuesUnfixed=X issuesFixed=Y unfixed={...}` after each module's QA; `[COURSE-MERGE] modules=N totalSlides=M fallbackModules=[...] dedupeRemoved=K` after global dedupe. **Renderer untouched** (per architect: "não refatore o renderer existente").
- **v5.2.4** — Surgical truncation/typo pass on top of v5.2.3. **Truncation verbs expanded**: added indicative/1ª-pessoa-plural forms (`Utiliza`/`Usamos`/`Aplicamos`/`Definimos`/`Capturamos`/`Garantimos`/`Buscamos`/`Enviamos`/etc) to the verb+`e`+prep detector AND to the leading verb+para detector. **3 new patterns**: (a) `verb + e + prep` no-subject — "Utiliza e para garantir robustez."; (b) clause-initial verb-para after `.:;` — "...usamos para definir..."; (c) trailing `como e <noun>.` / `como, e '<short>'.` — "como e idade.", "como, e 'a'.". **Generic-gerund objective**: `isGenericObjective` now matches "<verb> trabalhando com <module-title>" and 5 other gerund glues (estudando/explorando/atuando em/atuando com/lidando com) — catches "Compreender trabalhando com JSON e APIs Web." **Python API typo repair** (`repairPythonApiTypos`): deterministic substitutions `super().init(` → `super().__init__(`, `def init(` → `def __init__(`, `<inst>.init(` → `<inst>.__init__(`, `__init_(`/`_init__(` → `__init__(`. Runs UNCONDITIONALLY on every code snippet (not only when validation fails) so even otherwise-valid snippets get the typo fixed.
- **v5.2.3** — Semantic consolidation pass. New issue types: `BROKEN_SEMANTIC_SENTENCE` (ends-with-bare-`e`/`ou`, "X: Verb." after colon, empty `(, )` parens, lone verb, `prep+verb.`), `MODULE_OBJECTIVE_VIOLATION` (per-kind FORBIDDEN map enforced ONLY on objective-class slides — `module_cover`/`takeaways`/`summary`/`closing` — so M8 "Boas Práticas" rejects "expressões e atribuições"/"entrada e saída com variáveis"/etc), `INCOMPLETE_CODE_SNIPPET` (single bare assignment, empty def/class body, multi-line w/o observable + no structure, dangling `=,(`). Repairs: `repairModuleObjective` injects 2-3 module-aligned fallbacks per kind (PEP8/venv/pip/docstring/logging/deploy for `best_practices`, etc.) when a cover/takeaways slide is emptied; `repairCodeSnippet` appends `print(<lhs>)` for single-line bare assignments and `pass` for empty def/class. All 3 new types added to `SEMANTIC_BLOCKERS` in per-module gate (index.ts).
