3. Código superficial/incompleto

sem conclusão lógica.

O planner está criando snippets incompletos ou pouco demonstrativos.

AÇÃO OBRIGATÓRIA

1. Criar validação forte de objetivos por módulo

Criar:
validateModuleObjectives(moduleTitle, objectives)

O objetivo deve ser semanticamente coerente com o módulo.

Exemplos:

Módulo 1:
permitido:
- variáveis
- tipos
- operadores
- input/output

Módulo 8:
permitido:
- PEP8
- venv
- pip
- setup.py
- estrutura de projeto
- docstrings
- logging
- deployment
- organização

proibido:
- variáveis básicas
- operadores
- input
- print
- tipos primitivos

Se objetivo violar domínio:
- issue FATAL;
- repairModuleObjectives();
- fallback por módulo;
- ou veto.

2. Criar repairModuleObjectives()

Exemplo:

ruim:
"Criar expressões e atribuições corretas em Python."

bom:
"Aplicar boas práticas de organização e manutenção de projetos Python."

ruim:
"Aplicar entrada e saída com variáveis."

bom:
"Gerenciar dependências e ambientes virtuais com venv e pip."

3. Endurecer detector de frases truncadas

Criar:
detectBrokenSemanticSentence()

Detectar:
- termina com "e"
- termina com ","
- termina com ":"
- termina com "usar."
- termina com "com."
- termina com "(, )"
- termina com "Que X?"
- frases sem objeto técnico
- frases sem verbo útil
- frases com placeholders implícitos

Exemplos inválidos:
- "Acessar Membros: Usar."
- "Usar modos de abertura, e 'a'."
- "Realize leitura (, )..."
- "Que Boas Práticas de Codificação?"

Adicionar:
BROKEN_SEMANTIC_SENTENCE
ao HARD_CRITICAL_TYPES.

Nenhum slide com frase quebrada pode exportar.

4. Melhorar geração de snippets de código

Criar:
validateCodeSnippet()

Regras:
- snippet precisa demonstrar começo + ação + resultado;
- snippet não pode terminar abruptamente;
- snippet deve ser executável/coerente;
- snippet precisa ter propósito pedagógico claro.

Exemplo inválido:
soma = sum(dados)

Exemplo válido:
soma = sum(dados)
logging.info(f"Soma calculada: {soma}")
return soma

5. Criar repairCodeSnippet()

Se snippet:
- estiver incompleto;
- terminar abruptamente;
- não demonstrar finalidade;
- tiver menos de X linhas úteis;

Então:
- completar semanticamente;
- adicionar fechamento coerente;
- adicionar exemplo mínimo funcional.
