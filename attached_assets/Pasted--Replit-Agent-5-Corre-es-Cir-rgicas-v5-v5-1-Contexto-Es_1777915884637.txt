# Replit Agent — 5 Correções Cirúrgicas (v5 → v5.1)

> **Contexto**: Este é um arquivo TypeScript de Supabase Edge Function que gera PPTX
> via PptxGenJS. As 5 correções abaixo são **substituições literais de texto** —
> cada bloco mostra exatamente o que procurar (DE) e o que colocar no lugar (PARA).
> Não interpretar, não reformular. Substituir o texto exato.

## CORREÇÃO 1 — Sanitizar `\n` literal nos items do Gemini

**Problema**: O Gemini gera itens com `\n` como dois caracteres literais (barra + n),
não como quebra de linha. A função `san()` não remove isso. Resultado: textos como
"elaboraria:\n\n1" aparecem nos slides.

**Localizar** o bloco de mapeamento de `rawSlides` dentro de `generateModuleSlides()`.
É o trecho que contém `items: Array.isArray(s.items)`.

**Substituir DE:**
```typescript
      items: Array.isArray(s.items)
        ? s.items.slice(0, 6).map((x: any) => String(x).slice(0, 110))
        : [],
```

**PARA:**
```typescript
      items: Array.isArray(s.items)
        ? s.items.slice(0, 6).map((x: any) =>
            String(x).replace(/\\n/g, " ").replace(/\\t/g, " ").trim().slice(0, 110)
          )
        : [],
```

**Também substituir DE** (leftItems):
```typescript
      leftItems: Array.isArray(s.leftItems)
        ? s.leftItems.slice(0, 4).map((x: any) => String(x).slice(0, 90))
        : undefined,
      rightItems: Array.isArray(s.rightItems)
        ? s.rightItems.slice(0, 4).map((x: any) => String(x).slice(0, 90))
        : undefined,
```

**PARA:**
```typescript
      leftItems: Array.isArray(s.leftItems)
        ? s.leftItems.slice(0, 4).map((x: any) =>
            String(x).replace(/\\n/g, " ").replace(/\\t/g, " ").trim().slice(0, 90)
          )
        : undefined,
      rightItems: Array.isArray(s.rightItems)
        ? s.rightItems.slice(0, 4).map((x: any) =>
            String(x).replace(/\\n/g, " ").replace(/\\t/g, " ").trim().slice(0, 90)
          )
        : undefined,
```

## CORREÇÃO 2 — Linha divisória no `header()`: full-width → acento curto

**Problema**: A `rect` no final da função `header()` percorre todo o slide (`w: CW`),
criando uma linha cinza que corta visualmente cada slide de conteúdo — o padrão
mais característico de "AI template".

**Localizar** a função `header()`. No final dela, encontrar o `addShape` da linha
divisória. É o único `addShape` com `fill: { color: d.border }` na função `header()`.

**Substituir DE:**
```typescript
  slide.addShape("rect" as any, {
    x: ML,
    y: CONTENT_Y - 0.06,
    w: CW,
    h: 0.025,
    fill: { color: d.border },
  });
```

**PARA:**
```typescript
  slide.addShape("rect" as any, {
    x: ML,
    y: CONTENT_Y - 0.05,
    w: 0.4,
    h: 0.035,
    fill: { color: d.accent },
  });
```

## CORREÇÃO 3 — `renderProcess()`: caixas altas demais → altura compacta centralizada

**Problema**: As caixas de processo ocupam toda a altura do slide (`boxH = areaH = 5.21"`),
mas o conteúdo (número + 2 linhas de texto) ocupa apenas ~1.4". Resultado: metade inferior
de cada caixa completamente vazia em todos os slides de processo.

**Localizar** a função `renderProcess()`. Encontrar as linhas que definem `areaY` e `areaH`
e `boxH`. São as primeiras linhas depois do `if (items.length === 0)`.

**Substituir DE:**
```typescript
  const n = items.length;
  const areaY = CONTENT_Y + 0.25;
  const areaH = FOOTER_Y - areaY - 0.25;
  const arrowW = n <= 4 ? 0.28 : 0.2;
  const totalArrows = (n - 1) * arrowW;
  const boxW = (CW - totalArrows) / n;
  const boxH = areaH;
```

**PARA:**
```typescript
  const n = items.length;
  const arrowW = n <= 4 ? 0.28 : 0.2;
  const totalArrows = (n - 1) * arrowW;
  const boxW = (CW - totalArrows) / n;
  // Altura compacta: suficiente para badge + texto + respiro, sem desperdício
  const boxH = n <= 3 ? 2.4 : n === 4 ? 2.2 : 2.0;
  // Centralizar verticalmente no espaço de conteúdo
  const areaY = CONTENT_Y + (CONTENT_H - boxH) / 2;
  const areaH = boxH; // mantido por compatibilidade com o restante do código
```

## CORREÇÃO 4 — Paleta `default`: trocar cores genéricas por azul marinho institucional

**Problema**: A paleta `default` usa indigo genérico (`4F46E5`) idêntico ao v4.
Como é a paleta usada pela maioria dos usuários, o deck parece sempre o mesmo.

**Localizar** o `PALETTE_MAP`. É o objeto `const PALETTE_MAP: Record<string, ...>`.

**Substituir apenas a linha da entrada `default` DE:**
```typescript
  default:    ["4F46E5", "7C3AED", "0891B2", "F59E0B", "0F0F1A"],
```

**PARA:**
```typescript
  default:    ["1E3A5F", "2E6DA4", "C47F17", "E8A020", "0A1628"],
```

> As 5 posições são: [accent_primário, accent_secundário, accent_terciário, highlight, coverBg]
> Azul marinho (`1E3A5F`) como cor dominante, azul médio (`2E6DA4`) como secundário,
> dourado-âmbar (`C47F17` e `E8A020`) como acentos de destaque. Cover escuro (`0A1628`).

## CORREÇÃO 5 — Substituir strings "v4" residuais no HTTP handler

inconsistente com ENGINE_VERSION 5.0.0.

**Localizar** o bloco de geração do `fileName` (procurar por `PPTX-v4`).

**Substituir DE:**
```typescript
    const fileName = `${userId}/${safeName}-PPTX-v4-${dateStr}-${ts}.pptx`;
```

**PARA:**
```typescript
    const fileName = `${userId}/${safeName}-PPTX-v5-${dateStr}-${ts}.pptx`;
```

**Localizar** os dois `console.log` com `[V4-WRITE]` e substituir ambos:

**DE:**

```typescript
    console.log(

    );
```
**PARA:**
```typescript
    console.log(
      `[V5-WRITE] raw_bytes=${rawBytes.byteLength} | magic=${rawBytes[0]}_${rawBytes[1]}_${rawBytes[2]}_${rawBytes[3]}`,
    );
```

**DE:**
```typescript
    console.log(
      `[V4-WRITE] repaired_bytes=${pptxData.byteLength} slides=${repairDiag.slide_count}`,

```
**PARA:**
```typescript
    console.log(
      `[V5-WRITE] repaired_bytes=${pptxData.byteLength} slides=${repairDiag.slide_count}`,
```

**Localizar** o `usage_events` insert e substituir DE:
```typescript

```
**PARA:**
```typescript
        event_type: "COURSE_EXPORTED_PPTX_V5",
```

**Localizar** o JSON de resposta e substituir DE:
```typescript

```
**PARA:**
```typescript

```

## Verificação obrigatória após aplicar

Após todas as substituições, fazer uma busca textual no arquivo por cada string abaixo.
**Nenhuma deve aparecer:**

- `PPTX-v4` → deve ser `PPTX-v5`
- `V4-WRITE` → deve ser `V5-WRITE`
- `PPTX_V4` → deve ser `PPTX_V5`
- `version: "v4"` → deve ser `version: "v5"`
- `w: CW,` dentro da função `header()` → não deve mais existir
- `const boxH = areaH;` como linha isolada em `renderProcess()` → não deve mais existir
- `4F46E5` → não deve mais existir no PALETTE_MAP
- `.slice(0, 110)` sem `.replace(/\\n/g` antes → não deve mais existir nos items

**O que NÃO alterar:**
- Qualquer outra entrada do `PALETTE_MAP` além de `default`
- A função `repairPptxPackage()`
- A lógica de autenticação
- Qualquer renderer além de `renderProcess()` e `header()`
- A constante `ENGINE_VERSION` (já está em `"5.0.0"`)
