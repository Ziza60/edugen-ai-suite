1. CRIAR CODE COMPLETENESS GUARD
========================================

Criar:

validateSemanticCodeCompleteness(slide)

Problema atual:
muitos slides de código terminam assim:
- "# ..."
- código truncado;
- função sem retorno;
- exemplo sem fechamento;
- snippet incompleto;
- pseudo-código disfarçado de código real.

Exemplos ruins atuais:
- slide 13
- slide 37
- slide 42

Regras:

Marcar como INCOMPLETE_CODE_SEMANTIC quando:
- termina com "# ..."
- termina com comentário aberto;
- termina imediatamente após lógica parcial;
- função sem fechamento lógico;
- classe sem uso;
- snippet muito curto para o título prometido;
- exemplo não demonstra o conceito anunciado;
- há setup sem output/result;
- exemplo não tem conclusão semântica.

Exemplo ruim:
```python
if total > 50:
    desconto = total * 0.05
# ...
Exemplo bom:
if total > 50:
    desconto = total * 0.05

totalfinal = total - desconto
return totalfinal
